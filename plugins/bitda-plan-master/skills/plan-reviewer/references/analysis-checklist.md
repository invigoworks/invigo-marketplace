# 기획 검토 체크리스트

## 1. 논리적 일관성 (Logical Consistency)

### 1.1 모순 검사
- [ ] 동일 조건에 대한 다른 결과가 명시되지 않았는가?
- [ ] 상호 배타적 조건이 명확히 구분되어 있는가?
- [ ] 시간 순서가 논리적인가? (미래 데이터를 과거에 참조하지 않는가)
- [ ] 수량/금액 계산이 일치하는가?

### 1.2 인과관계
- [ ] 원인 → 결과 연결이 명확한가?
- [ ] "왜?"라는 질문에 답할 수 있는가?
- [ ] 숨겨진 가정이 없는가?

### 1.3 용어 일관성
- [ ] 같은 개념에 같은 용어를 사용하는가?
- [ ] 약어가 최초 사용 시 정의되어 있는가?
- [ ] 도메인 용어가 정의되어 있는가?

### 1.4 범위 일관성
- [ ] 기획 범위가 명확히 정의되어 있는가?
- [ ] Out of scope 항목이 명시되어 있는가?
- [ ] 관련 기능과의 경계가 명확한가?

---

## 2. 누락 케이스 (Missing Cases)

### 2.1 엣지 케이스
- [ ] 빈 값 / null 처리
- [ ] 최소값 / 최대값 경계
- [ ] 0개, 1개, 다수 케이스
- [ ] 특수문자, 이모지, 긴 텍스트

### 2.2 에러 상황
- [ ] 네트워크 에러
- [ ] 서버 에러 (500)
- [ ] 권한 에러 (403)
- [ ] 인증 에러 (401)
- [ ] 데이터 없음 (404)
- [ ] 유효성 검사 실패
- [ ] 타임아웃

### 2.3 동시성
- [ ] 동시 수정 충돌
- [ ] 중복 요청 (버튼 연타)
- [ ] 경쟁 상태 (race condition)
- [ ] 락(lock) 필요 여부

### 2.4 상태 전이
- [ ] 모든 상태가 정의되어 있는가?
- [ ] 각 상태에서 가능한 액션이 정의되어 있는가?
- [ ] 불가능한 전이가 명시되어 있는가?
- [ ] 되돌리기/취소가 가능한가?

### 2.5 데이터 유효성
- [ ] 필수값 검증
- [ ] 포맷 검증 (이메일, 전화번호 등)
- [ ] 범위 검증 (최소/최대)
- [ ] 관계 검증 (참조 무결성)

## 3. 의존성 (Dependencies)

### 3.1 선행 조건
- [ ] 이 기능을 사용하기 위한 전제 조건이 명시되어 있는가?
- [ ] 필수 데이터가 어디서 오는지 정의되어 있는가?
- [ ] 필요한 권한/역할이 명시되어 있는가?
- [ ] 선행 작업 완료 여부 확인 방법이 있는가?

### 3.2 후행 영향
- [ ] 이 기능이 다른 기능에 미치는 영향이 파악되어 있는가?
- [ ] 데이터 삭제 시 참조하는 다른 데이터 처리
- [ ] 상태 변경 시 연쇄 업데이트 범위
- [ ] 롤백 시 영향 범위

### 3.3 외부 의존
- [ ] 외부 API/서비스 의존이 명시되어 있는가?
- [ ] 외부 서비스 장애 시 대응 방안이 있는가?
- [ ] SLA/가용성 요구사항이 정의되어 있는가?
- [ ] 외부 데이터 형식 변경 대응 방안이 있는가?

### 3.4 데이터 의존
- [ ] 필요한 데이터의 출처가 명확한가?
- [ ] 데이터 동기화 주기가 정의되어 있는가?
- [ ] 캐싱 전략이 정의되어 있는가?
- [ ] 데이터 정합성 유지 방안이 있는가?

## 4. 사용자 시나리오 (User Scenarios)

### 4.1 Happy Path
- [ ] 주요 성공 시나리오가 완전히 기술되어 있는가?
- [ ] 시작점 → 종료점이 명확한가?
- [ ] 각 단계가 구체적인가?
- [ ] 예상 결과가 명시되어 있는가?

### 4.2 Unhappy Path
- [ ] 실패 시나리오가 정의되어 있는가?
- [ ] 에러 메시지가 정의되어 있는가?
- [ ] 복구 방법이 제시되어 있는가?
- [ ] 사용자에게 안내가 제공되는가?

### 4.3 사용자 실수 대응
- [ ] 잘못된 입력 시 피드백
- [ ] 실수로 삭제 시 복구 방법
- [ ] 중복 제출 방지
- [ ] 실행 취소(Undo) 기능

### 4.4 중단/재진입
- [ ] 작업 중단 시 데이터 보존 여부
- [ ] 임시 저장 기능 필요 여부
- [ ] 재진입 시 상태 복원 방법
- [ ] 세션 만료 시 처리

### 4.5 접근성
- [ ] 다양한 디바이스 지원 (모바일, 태블릿)
- [ ] 키보드 네비게이션
- [ ] 스크린 리더 지원
- [ ] 색약/색맹 고려

## 5. 기술적 실현 가능성 (Technical Feasibility)

### 5.1 성능
- [ ] 예상 데이터 볼륨이 정의되어 있는가?
- [ ] 응답 시간 요구사항이 있는가?
- [ ] 대용량 데이터 처리 방안이 있는가?
- [ ] 페이지네이션/무한스크롤 필요 여부

### 5.2 보안
- [ ] 인증/인가 요구사항이 명확한가?
- [ ] 민감 정보 처리 방안이 있는가?
- [ ] 감사 로그 필요 여부
- [ ] 데이터 암호화 필요 여부

### 5.3 확장성
- [ ] 사용자 증가 대응 방안
- [ ] 데이터 증가 대응 방안
- [ ] 기능 확장 가능한 구조인가?
- [ ] 다국어/다통화 고려 여부

### 5.4 유지보수성
- [ ] 설정 가능한 항목이 하드코딩되지 않는가?
- [ ] 변경 가능성 높은 부분이 식별되어 있는가?
- [ ] 운영 모니터링 요구사항이 있는가?
- [ ] 로깅 요구사항이 정의되어 있는가?

## 6. 비즈니스 관점

### 6.1 목적 정합성
- [ ] 기능의 목적이 비즈니스 목표와 일치하는가?
- [ ] ROI가 합리적인가?
- [ ] 우선순위가 적절한가?

- [ ] 법적 요구사항 반영 여부 (개인정보보호법 등)

- [ ] 업계 표준 준수 여부

### 6.3 운영 고려
- [ ] 고객센터 대응 시나리오
- [ ] 장애 대응 절차
- [ ] 운영자 관리 기능

## 7. 문서 품질

### 7.1 완결성
- [ ] 필수 섹션이 모두 포함되어 있는가?
- [ ] 각 섹션이 충분히 상세한가?
- [ ] 첨부/참조 자료가 유효한가?

### 7.2 명확성
- [ ] 모호한 표현이 없는가?
- [ ] 구체적인 예시가 포함되어 있는가?
- [ ] 다이어그램/플로우차트가 적절히 사용되었는가?

### 7.3 추적성
- [ ] 요구사항과 기능의 매핑이 가능한가?
- [ ] 변경 이력이 관리되고 있는가?
- [ ] 관련 문서 참조가 되어 있는가?

## 검토 결과 분류 기준

### 심각도 (Severity)

| 등급 | 기준 | 조치 |
|------|------|------|
| **Critical** | 기능 동작 불가, 데이터 손실 가능, 보안 취약점 | 즉시 해결 필수 |
| **Major** | 주요 기능 영향, 사용자 경험 저하, 성능 문제 | 출시 전 해결 권장 |
| **Minor** | 사소한 불편, 개선 가능 사항, 문서 품질 | 백로그 등록 |

### 확신도 (Confidence)

| 등급 | 설명 |
|------|------|
| **확실함** | 명백한 문제, 논리적 모순 |
| **가능성 높음** | 높은 확률로 문제가 될 것 |
| **검토 필요** | 의도인지 누락인지 확인 필요 |
